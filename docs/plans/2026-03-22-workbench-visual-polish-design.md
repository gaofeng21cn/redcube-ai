# Workbench 视觉收口设计

日期：2026-03-22

## 目标

- 在不推翻现有信息架构的前提下，把 Workbench 外壳做得更现代、专业、有设计感
- 让首页在空工作区状态下既有明确动作，也有更像产品而不是后台的视觉层次
- 保持移动端可用，并补齐 reduced-motion 兜底

## 方案

### 1. 视觉语言

- 正文使用无衬线 UI 字体
- 大标题切到衬线 display 字体，形成更明显的产品感和编辑感
- 背景、页头和主面板采用多层渐变与轻微纹理，不走纯平白板风格

### 2. 组件收口

- 顶部 header 加装饰层和更清晰的视觉重心
- tabs 与按钮做成更像产品导航而不是朴素控制条
- metrics strip 从“胶囊标签”升级为小卡片
- onboarding / hero / 状态卡统一同一套面板语言

### 3. 动效与可访问性

- 页面切换加轻量淡入上移动效
- 保留 focus-visible
- 增加 `prefers-reduced-motion: reduce` 兜底

## 验收标准

- 顶部 shell、metrics、onboarding 与 hero 视觉明显统一
- 标题层级更强，正文信息更稳
- 测试覆盖视觉系统的关键 CSS 钩子
