# 公开 GitHub 仓库收口设计

日期：2026-03-22

## 目标

- 将当前仓库作为全新的公开 GitHub 仓库发布
- 仓库内不包含旧历史、旧工作目录痕迹、作者人设、私有 prompts 与真实业务素材
- 为公开使用者补齐最基本的上手、发布、私有层配置文档

## 设计决策

### 1. 公开仓库只保留功能层

保留内容：

- 源代码
- 公开安全 prompts
- 测试
- 文档

不保留内容：

- 业务 workspace
- 运行产物
- 私有作者信息
- 私有 prompts
- 历史项目数据

### 2. 私有信息统一放到仓库外

私有层的推荐落点：

- `~/.config/redcube/`
- `<workspace>/.redcube/`
- 本机临时覆盖：`config/local/`

其中 `config/local/` 在公开仓库内默认整体忽略，只保留占位与说明文件，降低误提交风险。

### 3. 文档分两层

- `README.md`：仓库首页说明，解释项目定位、目录结构、快速开始、私有层原则
- `docs/tutorials/*.md`：面向实际操作的教程，覆盖 GitHub 发布和私有层迁移

## 验收标准

- 仓库可以直接 `gh repo create ... --push`
- 文档中不出现旧私有目录、旧作者人设、旧项目名等敏感内容
- 文档足够说明如何把个人配置继续放在私有层
- 推送前有 fresh 验证记录
