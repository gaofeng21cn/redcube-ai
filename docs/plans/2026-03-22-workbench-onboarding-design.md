# Workbench 空状态与首轮启动引导设计

日期：2026-03-22

## 目标

- 优化 Workbench 在空工作区和新主题首轮启动时的界面引导
- 把“挂接工作区 -> 新建任务 -> 自动首轮生成”变成首屏可理解、可操作的主行动线
- 不改后端接口，不新增复杂状态机，优先复用现有前端动作与创建任务后自动开跑逻辑

## 当前问题

- 空工作区时只有一句提示，用户不知道下一步该做什么
- `新建任务` 的存在感主要在页头按钮，首屏正文没有明确承接
- 对“创建后会自动进入首轮生成”这件事，页面没有直接说明
- 已有主题但尚未出产物时，项目页的下一步提示仍偏摘要，不够像启动板

## 方案

### 1. 空工作区改成三步启动板

在 Workbench 与 Projects 的空状态都改为统一的 onboarding 区块，明确展示：

1. 连接工作区
2. 创建任务骨架
3. 自动首轮生成

每一步都给一句说明和明确动作入口。

### 2. 已有主题但尚未出产物时，增加首轮启动板

在项目总览顶部增加一个“首轮启动”卡片区块，解释：

- 当前是否已有输入
- 现在最建议做什么
- 创建后或补完输入后会发生什么

### 3. 交互复用现有动作

复用现有逻辑，不加新 API：

- `toggleWorkspaceBtn` 对应打开工作区编辑器
- `createTaskModal` 对应新建任务
- `createTask()` 成功后自动 `startTopicWorkflow()`
- `refresh-overview` 继续刷新工作区

## 验收标准

- 空工作区首屏出现三步启动引导
- 能从正文区直接触发“更换工作区”和“新建任务”
- 项目总览在首轮生成前显示明确启动建议
- 测试覆盖新的引导文案与动作入口
