# Workbench Stitch 合并设计

日期：2026-03-24

## 目标

- 将 Stitch 导出方案中“第二个”的视觉系统与“第三个”的信息架构合并，落到 RedCube AI Workbench 的真实首页流程里
- 让首页从“更好看的后台”升级为“内容生产导演台”
- 保持现有真实功能链路不变：
  - 连接工作区
  - 新建任务
  - 首轮生成
  - 查看运行状态
  - 后续局部重跑

## 范围

本轮只处理首页相关的三个场景：

- 空工作区首页
- 首轮启动页
- 项目总览首页的 hero / 状态摘要

不在本轮范围内：

- 设置页重做
- 运行页重构
- 项目列表页整体重排
- 后端 API 或工作流逻辑变更

## 设计决策

### 1. 视觉系统采用“第二个”

保留第二个方案的这些核心特征：

- 暖白底，不用冷白和大面积青灰科技底
- `Newsreader` + `Manrope` 的编辑型字体组合
- sage 绿作为主强调色，而不是更强烈的 teal 科技色
- “No-Line” 分区原则：少用硬边框，多用色块层级与留白分组
- 更克制的圆角、阴影和材质，让页面像高端编辑工作室而不是通用 SaaS 仪表盘

明确不直接照搬第二个的部分：

- 作品集式大图卡片墙
- 过于 boutique agency 的内容语义
- 与当前 Workbench 功能无关的占位导航和 dataset / deployment 等通用 AI 后台信息

### 2. 信息架构采用“第三个”

保留第三个方案的这些结构优点：

- 首屏先展示当前状态，再展示可执行动作
- Hero 区直接承担“当前导演任务 / 下一步动作 / 当前系统状态”
- 状态摘要位于首屏主叙事之下，而不是分散在页头零散指标里
- 页面从上到下形成明确的导演台工作流：
  - 当前环境
  - 当前任务
  - 当前阶段状态
  - 最近运行 / 当前产物方向

明确不直接照搬第三个的部分：

- 偏冷的 frosted tech 玻璃质感
- Space Grotesk + teal 的 generic AI dashboard 气质
- 大面积“实验室控制台”语义

### 3. 合并后的定位

合并后的首页不叫“科技控制台”，也不叫“设计 agency 展示页”，而是：

**Editorial Director Console**

它应该传达的是：

- 这是一个高质量图文笔记生产工作台
- 系统有明确的阶段感和执行链路
- 用户在首页就能知道现在该做什么
- 视觉上有编辑感、专业感和产品感，而不是模板化后台感

## 页面结构

### A. 空工作区首页

改造目标：从“三步说明书”升级为“导演台首屏”。

结构改为四段：

1. Hero 主区
- 左侧：标题、当前场景说明、主 CTA
- 右侧：当前导演任务卡
- 只保留一个主动作：
  - 优先为“连接工作区”或“新建任务”，取决于当前状态

2. 状态摘要带
- 显示：
  - 工作区接入状态
  - 当前可执行阶段
  - 是否具备首轮启动条件
  - 当前是否已有运行记录

3. 行动卡组
- 三张卡，不再是教程步骤，而是导演台动作：
  - 连接工作区
  - 创建任务
  - 查看运行 / 首轮机制

4. 系统判断提示
- 用一块更像“导演建议”的 callout
- 文案口吻从说明式改成判断式，例如：
  - 当前缺少真实工作区，因此还不能进入生成阶段
  - 接入工作区后，系统会先识别主题，再允许创建或进入已有任务

### B. 首轮启动页

保留“先跑第一轮再细修”的逻辑，但改成更像项目启动板：

1. 顶部 Hero
- 当前主题名称
- 当前状态标签
- 当前最建议动作

2. 右侧或并排状态卡
- 当前导演任务
- 输入准备度
- 首轮生成是否可启动

3. 下方三张行动卡
- 补齐输入材料
- 开始首轮生成
- 查看最近运行

这里仍保留现有行为，但视觉和文案要从“步骤 1/2/3”变成“启动板 / 导演动作”。

### C. 项目总览首页

当前 `renderTopicHeroSection` 和 `renderWorkbenchOverviewView` 已经有基础，但还不够“导演台”。

调整为：

1. 项目 Hero
- 左侧：项目名、项目叙事、当前建议动作
- 右侧：当前运行状态摘要卡

2. 状态摘要卡组
- 输入材料数
- 已生成笔记数
- 最近运行状态
- 可局部重跑提示

3. 最近动作 / 当前方向提示
- 不是单纯展示数字
- 而是告诉用户：
  - 现在优先看什么
  - 是补输入还是看产物
  - 是重新首轮，还是做局部重跑

## 组件与样式策略

### 1. Header

- 保留顶部导航功能
- 降低 header 内工作区信息与空指标盒子的占比
- 让真正的页面主叙事回到正文 hero，而不是停留在页头

### 2. Hero

- 用第二个方案的 editorial hero 语言
- 大标题 serif
- 行动按钮更克制，主次分明
- 使用暖白与 sage 分层，不使用强科技玻璃感

### 3. 状态卡

- 采用第三个方案的信息分块逻辑
- 但样式采用第二个的纸感 / 暖白分层
- 卡片之间用留白和浅色层次区分，不用重边框

### 4. Overview / Metrics

- 现有 metrics strip 需要降级
- 将“空数字盒子”替换为“可判断的状态摘要”
- 指标要服务决策，而不是只服务装饰

### 5. 文案语气

统一改成：

- 判断式
- 导演台式
- 面向生产动作

避免：

- 教程说明书口吻
- 泛 AI 平台措辞
- 与真实工作流无关的“炫技型” dashboard 文案

  - 新首页结构与视觉系统断言

## 验收标准

- 首页视觉明显更接近第二个的编辑质感，而不是当前偏产品后台的混合风格
- 首页信息顺序明显更接近第三个的导演台逻辑
- 空工作区状态下，不再出现按钮重复和主线分散的问题
- 首轮启动与项目总览都能在正文首屏直接给出“当前最该做什么”
- 不影响现有真实功能链路
- 浏览器实测下，桌面端和移动端都可用
