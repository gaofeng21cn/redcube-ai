# 文档治理规则

## 目标

这份治理说明用于约束 `RedCube AI` 仓库的 `docs/` 体系，确保对外公开面稳定双语、对内操作面保持中文清晰，并避免无意义的中英混杂。

## 核心规则

1. 对外文档必须同时提供英文 `.md` 与中文 `.zh-CN.md`，并在改动时同步更新。
2. 内部技术、规划、备忘与操作文档默认使用中文，除非明确提升到双语公开面。
3. 专业术语可以保留英文，但不要在普通叙述里无意义地插入英文表达。
4. `docs/README*` 应保持统一结构，清楚区分双语公开面、内部资料与治理入口。

## 工作方式

- 新增公开文档前，先确认它是否真的属于默认公开面，再同步准备中英双语镜像。
- 内部文档通常只需要更新中文版本；若未来要升级为公开面，再补写英文镜像。
- 反复出现的术语与归类规则优先写在本文件中，其他文档引用即可，避免重复表述。
