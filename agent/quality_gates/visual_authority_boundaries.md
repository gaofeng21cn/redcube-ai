# Visual Authority Boundaries

Owner: RedCube AI
Purpose: shared quality gate for all RCA standard OPL Agent stages.

Gate rules:
- Visual truth owner: `redcube_ai`
- Communication and visual direction verdict owner: `redcube_ai`
- Review and export verdict owner: `redcube_ai`
- Artifact mutation authorization owner: `redcube_ai`
- Visual memory accept/reject owner: `redcube_ai`
- Owner receipt signer: `redcube_ai`
- Visual pack discipline stays in RCA: image-first, HTML, native PPTX, screenshot review, export, artifact locator, visual memory, and native helper implementation are domain quality surfaces, not generic runtime residue.
- Production evidence tail must produce artifact-producing owner receipt, review/export receipt, visual memory receipt, workspace receipt scaleout, production-like no-regression ref, Temporal controlled visual-stage long-soak ref, human review receipt, or RCA typed blocker.

Forbidden substitutions:
- Provider completion cannot become visual ready, exportable, handoffable, source-ready, or review-pass verdict.
- File presence cannot become artifact acceptance.
- Mechanical screenshot metrics cannot replace director review.
- Queue, retry, lifecycle, or session state cannot become domain quality verdict.
- OPL generated surfaces cannot write visual truth, artifact bodies, memory bodies, review/export verdicts, or owner receipts.
- Refs-only accounting, session currentness, workbench projection, naming cleanup, or compatibility guard cannot become production evidence progress.
