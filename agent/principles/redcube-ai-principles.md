# RedCube AI Principles

Owner: `redcube-ai`
Purpose: visual-deliverable specialization of the OPL standard-agent AI-first principle pack.
State: `active_domain_specialization`
Machine boundary: this is a human-readable domain specialization. Machine-readable adoption is in `contracts/standard-agent-principles-adoption.json`; visual truth and executable authority remain in RCA contracts, source, receipts, artifacts, and validation outputs.

RCA adopts the OPL principles as a visual-deliverable agent:

- Intake is the `source_intake` stage and its prompt, not a standalone Skill. It should produce source/material refs, route-back refs, human gates, typed blockers, or owner receipts before downstream visual work moves.
- Communication strategy, visual direction, artifact creation, review/revision, export, package handoff, visual memory accept/reject, and artifact mutation authorization remain RCA-owned judgments.
- Screenshot checks, render inspection, schema completeness, generated surfaces, and package consistency checks are evidence or blockers. They do not replace AI-first visual review, RCA export verdicts, owner receipts, or human review.
- `agent/stages/`, `agent/prompts/`, `agent/skills/`, `agent/professional_skills/`, `agent/quality_gates/`, `agent/knowledge/`, and `agent/tools/domain_affordances.md` are the declarative visual pack. Stage prompts run the stage, `agent/skills/*.md` holds stage policy refs, `agent/professional_skills/*/SKILL.md` holds reusable professional methods, and tools/helpers materialize refs and enforce boundaries without becoming a second visual truth source.
- OPL generated interfaces may invoke and project RCA refs. They cannot mutate canonical artifacts, accept visual memory, or issue review/export verdicts without RCA authority.

This specialization keeps RCA's visual loop AI-first while allowing OPL to host, recover, and project the stage shell.
