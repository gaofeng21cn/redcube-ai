# RedCube AI Domain Specialization

Owner: `redcube-ai`
Purpose: visual-deliverable specialization of the OPL standard-agent AI-first principle pack.
State: `active_domain_specialization`
Machine boundary: human-readable domain specialization. Machine-readable adoption is in `contracts/standard-agent-principles-adoption.json`; RCA contracts, source, receipts, and verification outputs remain authoritative for RCA behavior.

RCA adopts the OPL principles as a visual-deliverable Foundry Agent:

- Intake is the `source_intake` stage and its prompt, not a standalone Skill. It turns source material and work-order context into source manifests, visual material refs, owner questions, typed blockers, route-back refs, or handoff packets.
- RCA owns visual truth, communication strategy, visual direction, artifact authority, review/export verdicts, visual memory accept/reject decisions, owner receipts, and typed blockers.
- OPL may host the stage runtime, generated surfaces, refs-only workspace/source locator, and conformance readbacks. It does not write RCA visual truth, mutate canonical artifacts, sign owner receipts, create RCA typed blockers, or authorize review/export readiness.
- Visual strategy and artifact creation remain AI-first expert work. Scripts, native helpers, and contracts materialize refs, compare structure, and enforce authority gates; visual quality and export closeout remain RCA-owner gated.
- Generated interfaces and Agent Lab evidence are readback and handoff surfaces. They cannot claim visual ready, export ready, domain ready, or production ready without the required RCA owner evidence.

This specialization keeps RCA visual work AI-first while preventing OPL intake, professional skill packaging, or generated descriptors from becoming visual authority.
