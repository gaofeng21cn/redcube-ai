# Visual Deliverable Authoring Skill Policy

Owner: RedCube AI
Purpose: canonical skill policy for AI-first PPT deck, Xiaohongshu note, and poster-style visual deliverable stages.

Applies to:
- `source_intake`
- `communication_strategy`
- `visual_direction`
- `artifact_creation`
- `review_and_revision`

Policy:
- Use RedCube route contracts and approved source truth as the authoring contract.
- Keep visible deliverable text audience-facing; internal route names, source ids, prompt fields, and operator notes stay private.
- Treat screenshots and rendered pages as primary review evidence.
- Return refs, receipts, typed blockers, and repair targets instead of generic success flags.
