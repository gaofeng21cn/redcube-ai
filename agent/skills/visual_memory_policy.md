# Visual Memory Skill Policy

Owner: RedCube AI
Purpose: define how visual pattern memory proposals are generated, accepted, rejected, and exposed to OPL.

Policy:
- Memory bodies and accept/reject verdicts belong to RCA.
- Review stages may propose memory writeback refs when a visual pattern is proven by rendered artifacts and review evidence.
- Acceptance or rejection requires AI-first RCA judgment and owner receipt refs.
- OPL receives locator, proposal, receipt, and operator projection refs only; it cannot read or write memory bodies.
