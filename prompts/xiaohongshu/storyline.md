# xiaohongshu / storyline

## Goal

Turn current research/source truth into a reader-centered single-note or series narrative decision.

## Good Work

- Read the complete accepted source before choosing `single` or `series`; titles, folder names, and planning signals are only hints.
- Define audience judgment, tension, why-now, memory hook, progression, and resolution from the source and reader task rather than a fixed narrative formula.
- Choose a series only when distinct reader jobs/evidence clusters cannot remain coherent in one note. Give each planned note a non-overlapping question, evidence scope, transition, and no-repeat boundary.
- Preserve medical evidence, uncertainty, and action/seek-care boundaries. Avoid promotional single-drug framing.
- Route source insufficiency or unsafe claims upstream; do not make later page planning or rendering compensate.

## Handoff

Return the storyline object required by the attached output contract. For series mode include an executable series architecture; for single mode keep it explicitly not applicable.
