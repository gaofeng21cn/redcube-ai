# 单篇策划质控门禁（无CLI）

## 目的
- 在进入 HTML 生成前，拦截“简版策划/简版大纲”导致的质量断崖
- 仅使用智能体自检与结构化留痕，不依赖脚本门禁

## 适用阶段
- 阶段3单篇策划完成后
- 阶段4 HTML生成前（强制前置）

## 强制检查项（全部必须通过）

1. 文档完整性（01_单篇策划.md）
- 必须包含：本篇定位、3个备选标题、正文、标签、核心信息点、医学补充（医学主题）、署名文案
- `core_points_count >= 3`

2. 大纲完整性（02_信息图大纲.md）
- 必须包含：视觉总控卡、visual_world_card、版式原型库、layout_plan页级总表、逐页规划
- `layout_plan_row_count == total_pages`
- `page_block_count == total_pages`
- 每页必填字段命中率 `required_field_coverage >= 0.95`
- 每页必须包含 `主视觉动作 + 动作落地原语`（至少2项）

3. 逐页深度
- `avg_keypoints_per_page >= 3`
- `short_page_count == 0`（短页定义：缺“页面目标”或“页面衔接”或“防同构说明”）
- 医学主题每页至少出现 1 条“阈值/流程节点/误区/行动”类信息

4. 结构多样性（策划级）
- `unique_layout_prototype_count >= 4`（总页数>=8时）
- `max_consecutive_same_layout <= 2`
- 相邻页结构差异说明覆盖率 `adjacent_diff_coverage == 100%`

4A. 动作落地可执行性（策划级）
- `action_primitive_defined_rate == 100%`
- `high_risk_action_page_count == 0`（高风险动作页定义：写了“评分条/仪表盘/流程图/对话板”但无对应原语）

5. 版面均衡前置约束（策划级）
- `bottom_half_module_defined_rate == 100%`（除封面/结尾外）
- 每页必须写明“版心填充策略”与“纵向平衡检查”
- 必须显式给出 `critical_pages`（默认 P2-P(n-1)）
- `critical_page_bottom_module_defined_rate == 100%`
- `critical_page_top_heavy_predicted_count == 0`（按大纲占位预测）

## 输出格式（强制留痕）

在 `03_HTML生成说明.md` 增加 “上游输入完整性门禁” 段，必须包含：
- `outline_gate_pass: true/false`
- `total_pages`
- `layout_plan_row_count`
- `page_block_count`
- `required_field_coverage`
- `avg_keypoints_per_page`
- `unique_layout_prototype_count`
- `max_consecutive_same_layout`
- `short_page_count`
- `bottom_half_module_defined_rate`
- `critical_pages`
- `critical_page_bottom_module_defined_rate`
- `critical_page_top_heavy_predicted_count`
- `action_primitive_defined_rate`
- `high_risk_action_page_count`
- `failed_rules`（若失败列出规则编号）
- `rewrite_action`（若失败，记录重写页与重写原因）

## 失败处理（强制）
- 任一检查项失败：不得进入 HTML 生成
- 必须先重写 `01_单篇策划.md` / `02_信息图大纲.md`，再重复门禁
- 禁止带着失败稿继续导出 PNG 或写发布文案
