# 发布文案质控门禁（无CLI）

## 目的
- 解决“文案能发但过短、过平、缺互动”的质量问题
- 在交付前拦截简化稿与模板腔

## 适用阶段
- 阶段6 生成 `04_发布文案.md` 与 `.txt` 时强制执行

## 强制检查项（全部必须通过）

1. 标题质量
- `title_count == 3`
- `max_title_len <= 20`
- `title_hook_coverage >= 2`（问题钩子/行动收益/避坑提醒）

2. 正文深度
- `body_char_count >= 220` 且 `body_char_count <= 420`
- `body_structure_coverage == 4/4`（开场钩子/事实价值/可执行动作/互动提问）
- `generic_phrase_ratio <= 0.35`（空泛句占比）

3. 互动与转化
- `comment_prompt_count >= 1`
- `interaction_question_count >= 2`
- `actionable_step_count >= 2`

4. 标签与合规
- `hashtag_count >= 5` 且 `hashtag_count <= 8`
- `banned_terms_hit_count == 0`
- `meta_instruction_leak_count == 0`

## 输出留痕（强制）

在 `04_发布文案.md` 增加“质控自检”段，至少包含：
- `title_count`
- `title_hook_coverage`
- `body_char_count`
- `body_structure_coverage`
- `comment_prompt_count`
- `interaction_question_count`
- `actionable_step_count`
- `hashtag_count`
- `banned_terms_hit_count`
- `meta_instruction_leak_count`
- `gate_pass`
- `rewrite_action`

## 失败处理（强制）

- 任一指标失败：不得写入最终 `.txt`
- 必须先重写标题与正文，再重复门禁
- 禁止交付“单段短文 + 5标签”的简化稿
