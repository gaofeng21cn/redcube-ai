# 人设自动路由规则

> 目标：根据输入材料主题，自动判断并调取对应作者档案。
> 单一优先级：用户当次明确指定 > 输入目录AGENT_PRESET偏好 > 全局默认预设 > 自动判断。

## 1. 输入来源

路由需要读取以下信息：
1. 用户当次指令中的明确要求（如“这次用医学知识编辑”）
2. `input/<目录>/AGENT_PRESET.md`（可选，允许自然语言）
3. 全局默认预设：`system/自动小红书/defaults/AGENT_PRESET.default.md`
4. 原始材料内容与标题关键词
5. 作者档案库：`system/自动小红书/作者档案库.md`

## 2. 路由优先级（强制）

1. 用户明确指定人设：直接使用
2. 输入目录预设明确写“固定某人设”：使用该指定
3. 输入目录预设明确写“自动判断”：进入自动判断
4. 无输入目录预设：读取全局默认预设（默认自动判断）
5. 其余场景：进入自动判断

说明：
- `AGENT_PRESET.md` 只表达“人设偏好”（自动或指定），不定义路由算法
- 路由算法唯一来源是本文件

## 3. 自动判断规则

### 3.1 领域判定

- 若核心主题命中医学域（医学/健康/生命科学/诊疗）
  - 默认选择：`medical_public`
- 否则
  - 默认选择：`general_public`

### 3.2 置信度分级

- 高置信度：主题集中且关键词明显，直接路由
- 中置信度：主题混合但主次明确，路由并记录理由
- 低置信度：主题冲突或信息不足，必须提问用户确认

## 4. 低置信度处理（禁止脑补）

当出现以下任一情况，必须先询问用户：
1. 医学与非医学内容占比接近，无法判断主线
2. 用户目标受众和故事基调缺失，且会影响表达方式
3. 预设与用户当次指令冲突

建议提问模板：
- “当前素材兼具A/B两类主题，你希望本系列主视角按哪条主线展开？”
- “故事基调未指定，建议在‘专业理性/叙事探索/轻松活泼’中确认一个”

## 5. 调用输出格式（建议）

路由完成后，在任务文档记录：

```text
人设路由结果：medical_public
路由方式：自动判断
判定依据：命中关键词（诊疗、病理、预后），主题为医学知识科普
置信度：高
```

## 6. 一致性检查（执行时必须记录）

在 `00_任务说明.md` 记录以下四项：
1. 路由输入来源（用户指令 / 输入预设 / 默认预设 / 自动判断）
2. 最终选中人设
3. 判定依据（关键词或用户明确要求）
4. 是否存在冲突及处理方式

## 7. 扩展映射建议

当新增账号后，可按“主题 -> profile_id”追加映射，例如：
- 财经投资 -> finance_researcher
- 法律科普 -> legal_explainer
- 职场管理 -> management_coach
