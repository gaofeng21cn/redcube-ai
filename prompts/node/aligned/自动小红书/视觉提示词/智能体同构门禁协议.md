# 智能体同构门禁协议（无CLI版）

## 1. 目标

把门禁从“卡死生成”改为“创作优先、门禁兜底”：
- 先保证页面有统一风格世界观与叙事节奏
- 再用最小必要门禁拦截模板化退化

适用范围：
- `system/自动小红书/视觉提示词/活泼版.md`
- `system/自动小红书/视觉提示词/专业版.md`
- `system/自动小红书/视觉提示词/主视觉动作落地词典.md`

## 2. 强制输出顺序

1. `visual_world_card`（视觉世界观卡）
2. `layout_plan` 页级总表
3. HTML 单文件代码
4. 智能体同构自检报告

不允许跳步，不允许仅交付最终 HTML。

## 3. 输入来源白名单（强制）

允许输入来源：
- `input/<当前任务>/` 原始素材
- `output/<当前系列>/<当前笔记>/01_单篇策划.md`
- `output/<当前系列>/<当前笔记>/02_信息图大纲.md`
- `system/自动小红书/defaults/html_export_shell.reference.html`
- 用户当次明确指定的 Demo 文件

禁止输入来源：
- `output/archives/` 下历史产物
- 当前任务无关的 `output/<其他系列>/` 成品

自检报告必须输出：
- `reference_source_list`
- `forbidden_source_hit_count`

硬阈值：
- `forbidden_source_hit_count == 0`

## 4. 最小结构门禁（强制）

保留轻量结构指标，只防止完全同构：
- `unique_layout_count`
- `max_consecutive_same_layout`
- `card_stack_ratio`

阈值：
- `unique_layout_count >= 3`
- `max_consecutive_same_layout <= 3`
- `card_stack_ratio <= 0.6`

说明：
- `card_stack_ratio` = 主要信息以“纵向同构卡片堆叠”为主的页面占比

## 4A. 反模板诊断信号（生成期，非硬阈值）

用于提前发现“看起来通过、视觉却很笨”的页面：
- `max_empty_zone_ratio`：单页最大连续留白占比
- `header_list_footer_repeat_count`：出现“顶部标题 + 列表主体 + 底部信息块”固定句式的页数
- `hero_anchor_coverage_ratio`：存在主视觉锚点（占版心约 1/4 以上）的页面占比
- `meta_instruction_leak_count`：页面正文中出现流程术语（如 `封面完整署名`/`结尾完整署名`/`layout_plan`）的次数

建议区间（不硬卡）：
- `max_empty_zone_ratio <= 0.35`
- `header_list_footer_repeat_count <= 2`
- `hero_anchor_coverage_ratio >= 0.7`
- `meta_instruction_leak_count == 0`

## 4B. 版面均衡门禁（强制）

用于解决“内容集中在上半部分”的视觉失衡：
- `bottom_half_content_ratio`：页面下半区实质内容占比（按页平均）
- `top_bottom_balance_delta`：上下半区内容占位差绝对值（按页平均）
- `top_heavy_page_count`：上半区明显过载页面数（`top:bottom > 70:30`）

通过阈值：
- `bottom_half_content_ratio >= 0.40`
- `top_bottom_balance_delta <= 0.20`
- `top_heavy_page_count <= 1`（总页数 <= 10）；
  当总页数 > 10 时，`top_heavy_page_count <= ceil(total_pages * 0.1)`

说明：
- 封面与章节过渡页可适度上重，但需在后续页面补偿
- 若不达标，优先重写上重页（把关键信息模块下沉到中下版心）

补充说明：
- 4A 属于生成期纠偏信号（建议区间）
- 4B 属于发布前硬门禁（必须通过）
- 流程术语渗出属于硬错误，必须清零后再交付

## 4C. 关键机制页零容忍门禁（强制）

用于解决“均值通过但关键页仍上重下空”：
- `critical_pages`：除封面/结尾/章节过渡页外的全部机制页（通常 P2-P(n-1)）
- `critical_top_heavy_page_count`：关键机制页中上重下空页数（`top:bottom > 70:30`）
- `critical_bottom_module_coverage`：关键机制页中“下半区实质模块”覆盖率
- `critical_balance_table`：逐页记录 `top/mid/bottom` 占位与下半区模块名

通过阈值：
- `critical_top_heavy_page_count == 0`
- `critical_bottom_module_coverage == 100%`
- `critical_balance_table` 不允许缺页

说明：
- 下半区实质模块指：流程节点、参数块、矩阵、对话板、执行清单、风险动作条等承载信息模块
- 仅装饰元素、空白占位、单纯页脚不计入下半区承载

## 4D. 主视觉动作落地门禁（强制）

用于解决“计划里有动作，成品仍像清单”的问题。

强制指标：
- `action_realization_rate`：`layout_plan` 中主视觉动作被真实落地的页面占比
- `semantic_action_match_fail_count`：主视觉动作语义与实现原语不匹配页数
- `diagram_primitive_page_count`：使用图形原语（轨道/箭头/仪表/网络/矩阵）的页数
- `plain_card_fallback_page_count`：被判定为“仅卡片堆叠替代动作”的页数

通过阈值：
- `action_realization_rate >= 0.85`
- `semantic_action_match_fail_count == 0`
- 当总页数 >= 8 时：`diagram_primitive_page_count >= 2`
- `plain_card_fallback_page_count <= 1`

判定依据：
- 按 `system/自动小红书/视觉提示词/主视觉动作落地词典.md` 执行动作-原语匹配
- `评分条/仪表盘/流程图/对话板` 命中时，必须出现对应原语，普通卡片不计入落地

## 5. 视觉质感门禁（强制）

必须输出四项 1-5 分自评（允许半分）：
- `narrative_cohesion_score`（叙事连贯）
- `visual_consistency_score`（视觉世界观一致）
- `rhythm_variety_score`（页间节奏变化）
- `anti_template_score`（非模板感）

通过条件：
- 四项均 >= 3.5
- 平均分 >= 4.0

如用户提供 Demo，额外输出：
- `demo_style_anchor_hit_count`
- `demo_style_anchor_notes`

建议阈值（非硬卡）：
- `demo_style_anchor_hit_count >= 3`

## 6. 每页骨架签名表（审计留痕）

自检报告必须附：
- `page_X: tag1>tag2>tag3>...>tagN`
- `page_X: 主视觉动作 -> 已实现原语`
- `page_X: top/mid/bottom 占位 + 下半区模块`
- `critical_balance_table` 必须以导出PNG可视结果为准，不得仅凭大纲估计

用途：人工审计“是否只是换皮”。

## 7. 不达标处理（强制）

若未通过：
1. 先定位最低分页面（通常 1-3 页）
2. 仅重写这些页面的结构与视觉动作
3. 保留已达标页面，避免整篇推倒重来
4. 重新输出自检报告后再交付
5. 若“反模板诊断信号”中有 2 项以上偏离建议区间，必须先完成针对性重写再交付
6. 若 `meta_instruction_leak_count > 0`，必须重写对应页面文案
7. 若 `action_realization_rate` 或 `semantic_action_match_fail_count` 不达标，必须优先重写不匹配页，禁止直接交付
8. 若 `critical_top_heavy_page_count > 0`，必须重写对应关键机制页，禁止进入发布文案阶段
9. 若 `critical_bottom_module_coverage < 100%`，必须先补齐下半区实质模块再导出 PNG

## 8. 与项目硬约束并行

本协议不替代以下硬约束，需同时满足：
- 3:4 画幅完整容纳，无溢出
- 禁止外链图片，仅使用 Font Awesome（Free）+ Emoji + CSS/SVG
- 对比信息上下分块（不做左右分栏对比）
- 署名规则：封面完整、结尾完整；内页默认不强制署名
