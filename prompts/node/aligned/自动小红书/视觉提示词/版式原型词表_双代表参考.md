# 版式原型词表（Demo双代表参考附录）

## 用途

该附录用于支持“先 layout_plan 再 HTML”的结构规划，帮助智能体从一开始就避免列表同构。

说明：
- 本附录只提供版式语法参考，不提供可直接复用的页面模板
- 必须结合当次大纲重写 DOM，禁止复制现成骨架

## 推荐版式原型（可扩展）

1. `LP_01_HERO_COVER`
- 语义：封面主视觉 + 价值点预告 + 完整署名
- 常见动作：中心爆点 / 背景气氛层 / 结论条前置
- 签名示例：`section>div>h1>p>ul>li>div`

2. `LP_02_CONFLICT_STACK`
- 语义：冲突对照（上下分块）+ 中间结论
- 常见动作：负向误区在上、正向方案在下
- 签名示例：`section>header>div>article>article>footer`

3. `LP_03_TIMELINE_FLOW`
- 语义：流程推进 / 节点串联
- 常见动作：纵向时间轴、节点编号、路径箭头
- 签名示例：`section>header>ol>li>li>li>aside`

4. `LP_04_CASE_ANALYSIS`
- 语义：案例拆解 / 场景复盘
- 常见动作：问题场景块 + 解释块 + 行动块
- 签名示例：`section>div>blockquote>div>ul>li`

5. `LP_05_MATRIX_COMPARE`
- 语义：矩阵化比较（禁止左右硬分栏）
- 常见动作：纵向多组卡片、组内结构统一
- 签名示例：`section>h2>div>div>ul>li>li`

6. `LP_06_SIGNAL_ALERT`
- 语义：红旗信号 / 风险警报
- 常见动作：警示头图 + 危险清单 + 就医动作
- 签名示例：`section>header>div>ul>li>li>footer`

7. `LP_07_CHECKLIST_ACTION`
- 语义：执行清单 / 复盘打卡
- 常见动作：勾选列表 + 优先级标签 + 下一步
- 签名示例：`section>header>ul>li>li>li>div`

8. `LP_08_CLOSING_CTA`
- 语义：结尾收束 + 行动召回 + 完整署名
- 常见动作：核心复盘三条 + CTA + 下期预告
- 签名示例：`section>div>h2>p>div>button>footer`

## 禁复用建议

1. 同一原型连续最多 2 页
2. `LP_08_CLOSING_CTA` 仅用于结尾页
3. 禁止把 `LP_07_CHECKLIST_ACTION` 扩散成全篇主骨架
4. 若连续两页都使用卡片组，第三页必须切换成流程/案例/信号原型

## 自检提示

1. 先统计 `layout_plan` 的原型分布
2. 再逐页比对签名前 N 标签序列
3. 若仅 class 或配色变化，判定为“未通过”
