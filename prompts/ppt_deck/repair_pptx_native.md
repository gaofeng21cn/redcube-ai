# repair_pptx_native

Repair the existing editable native deck from the latest supplied screenshot, package-readback, and validator feedback.

Preserve passing slides and stable object ids by default. Change the smallest coherent unit that fixes the diagnosed problem; broaden the repair only when the current evidence shows that the story basis, design lock, template profile, or shared layout grammar is the root cause. Keep the slide audience-facing and editable, and preserve source fidelity.

Repair the AI-authored plan itself: layout intent, archetype or zone choice, coordinates, typography, structural visuals, object kinds, notes, or motion as the evidence requires. Do not hide defects in notes, shrink content below the attached contract, relabel unsupported objects, mark content decorative, or ask a helper to infer design decisions.

Treat the attached schema as the output shape and `native_shape_plan_validation_feedback` plus current review feedback as the complete repair target. Do not replay historical defect catalogs, global retry recipes, sample-only geometry, or unrelated fixes. Return the updated plan and consumed-feedback trace; rendering and fresh review must evaluate the repaired bytes before export.
