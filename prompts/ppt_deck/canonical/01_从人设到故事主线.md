# 统一执行手册：人设 -> 主线 -> 大纲 -> 逐页 -> HTML -> PPTX

> 适用：当前讲者工作台下的任意讲课任务目录

## 最短可执行指令

```text
先读：素材库/03_装配规则/01_从人设到故事主线.md。
再读：素材库/03_装配规则/03_视觉生成执行手册（文档直驱+Skill增强）.md。
任务：<你的任务，例如“面向临床医生做一场AI辅助科研独立讲课，全链路产出到PPTX”>。
```

## 目录结构约定

以 `<讲课目录>` 表示目标目录：

```text
<讲课目录>/
├── 故事主线.md
├── 详细大纲.md
├── 大纲/
│   └── <章节文件>.md
├── 幻灯片/
│   └── <章节文件>.html
├── pptx/
│   └── <章节文件>.pptx
└── 参考材料/
```

命名说明：
- 单文件讲课：`<章节文件> = 讲课主题`（例如 `临床医生如何用AI做科研讲课`）
- 分章节讲课：`<章节文件> = PartX_主题`

## Step 0：选择执行模式（Step 4 前先定）

1. 模式A：文档直驱（默认）
2. 模式B：skill 增强（可选，调用 `complex-ppt-designer`）
3. 两种模式都使用同一视觉主提示词：`_skills/ppt-designer/prompts/02_generate_slides.md`

## Step 0.1：从零调研型任务分流判定（强制，先于 Step 1）

若本轮任务满足以下任一条件，不建议直接继续阅读本手册主体，而应先切换到：
- `素材库/03_装配规则/05_从零调研型讲课与PPT工作流.md`

触发条件：
1. 主题本身依赖最新公开资料才能成立
2. 现有 `素材库/` 与目标目录里没有现成证据底座
3. 用户明确要求“先做调研，再做讲课/PPT”
4. 任务的核心难点不在制片，而在“从零建立判断框架与证据链”

若不满足上述条件，继续按本手册执行。

## Step 0.5：命名判定（强制，先于 Step 1）

1. 先判断本次任务是“单文件”还是“分章节”
2. 判定规则：
   - 若只需产出 1 个完整 HTML，不做章节合并：使用单文件主题命名
   - 若需产出多个章节 HTML，或明确要后续合并：使用 `PartX_主题` 命名
3. 冲突优先级：用户本轮明确要求 > 本手册默认约定 > 历史命名习惯
4. 执行前必须回显一次命名决策：

```text
命名模式：单文件 / 分章节
章节文件名：<章节文件>
判定理由：<一句话>
```

## Step 0.6：目录补齐判定（强制，先于 Step 1）

1. 先检查 `<讲课目录>` 是否已有以下结构：
   - `故事主线.md`
   - `详细大纲.md`
   - `大纲/`
   - `幻灯片/`
   - `pptx/`
   - `参考材料/`
2. 若目标目录是旧任务目录或散文件目录，且缺少上述子目录，必须先补齐目录，再进入 Step 1。
3. 禁止在目标目录根部长期堆放“讲稿大纲 + html”而不回填标准五层结构。

## Step 1：人设 + 输入素材 + 任务要求 -> 故事主线

1. 选定主人设（只选一份，避免口径冲突）
2. 写清听众、时长、目标、成功标准
3. 用事实资产支撑关键转折，不做口号堆砌
4. 产出：`<讲课目录>/故事主线.md`

## Step 1.5：外部公开证据增强判定（按需，不强制）

1. 默认先用当前工作台现有素材与事实资产完成主线，不把“联网找证据”作为固定前置动作。
2. 只有在以下场景，才建议补外部公开证据增强说服力：
   - 面向医院管理层、跨专科或外部汇报，需要增强“为什么是现在”的判断
   - 涉及趋势、监管、政策、行业拐点、市场格局等时效性较强内容
   - 现有素材不足以支撑关键判断，且公开权威来源能显著补强可信度
3. 若不属于上述场景，或现有素材已足够，则可不补外部证据。
4. 一旦使用外部公开证据：
   - 优先官方文件、原始指南、正式机构报告、论文原文
   - 涉及时效性事实，必须写明日期
   - 统一沉淀到 `<讲课目录>/参考材料/外部证据补强.md` 或同类文件，不得只临时写进 HTML
   - 只补强关键判断，不把讲课做成“新闻汇总”

## Step 2：故事主线 + 输入素材 -> 详细大纲

1. 先按 Step 0.5 判定命名模式，再拆出结构和页数预算
2. 每页写清核心句、证据点、本页目标
3. 产出：`<讲课目录>/详细大纲.md`

## Step 3：详细大纲 -> 逐页设计（强制）

> 没有 `大纲/*.md`，不得进入 HTML 生成

1. 输出：
   - 单文件：`<讲课目录>/大纲/<讲课主题>.md`
   - 分章节：`<讲课目录>/大纲/PartX_主题.md`
2. 每页必须包含：标题、页面核心内容、视觉呈现方式、讲稿、过渡句
3. 逐页字段按：`素材库/03_装配规则/02_逐页设计与讲稿模板.md`
4. 本步若发现证据不一致，先回改 `详细大纲.md`

## Step 4：逐页设计 + 参考材料 -> HTML

1. 严格按 `大纲/*.md` 逐页生成 `幻灯片/*.html`
2. 视觉生成主提示词必须直接复用：`_skills/ppt-designer/prompts/02_generate_slides.md`
3. 只允许追加本工作台约束，不重写成熟主提示词
4. 禁止视觉模板路由（如 `renderSlide/layoutByType/cardsGrid/pageType`）
5. 每页在 `slidesData` 中必须是独立 `content` 代码块
6. 来源/署名/页眉页码按页控制，不做全局固定
7. 具体执行细节见：`素材库/03_装配规则/03_视觉生成执行手册（文档直驱+Skill增强）.md`

## Step 5：HTML -> PPTX

```bash
python3 tool/converter.py "<讲课目录>/幻灯片/<章节文件>.html"
```

批量补转：

```bash
cd "<讲课目录>/幻灯片"
python3 ../../tool/converter.py
```

## Step 6：完整课件合并（可选）

> 单文件讲课通常不需要本步；仅分章节或双版本任务使用

```bash
python3 tool/merger.py \
  --pptx-dir "<讲课目录>/pptx" \
  --name-prefix "<讲者名>" \
  --title-standard "<标准版标题>" \
  --title-general "<通用版标题>"
```

## 完成标准

1. `故事主线.md`、`详细大纲.md`、`大纲/*.md`、`幻灯片/*.html`、`pptx/*.pptx` 五层一致
2. 每一页都能在对应的 `大纲/*.md` 找到视觉方案与讲稿
3. HTML 页数、PPTX 页数、大纲页数一致
4. 讲课风格不是“全篇标题+列表”，而是多种布局语言混合
5. 本次执行模式（文档直驱/skill增强）与引用来源已明确记录
6. 若本轮使用外部公开证据，相关来源已沉淀到 `参考材料/`
