# 视觉生成与 AI 原生截图质控执行手册（文档直驱 + Skill增强）

> 目标：把 `大纲/*.md + *_视觉导演稿.md` 稳定转成高质量 HTML，并在转 PPTX 前完成 AI 原生截图质控与主动回修
> 核心原则：先做对，再检查；先复用成熟主提示词，再加本工作台约束

## 0. 固定主提示词（必须）

Step 4 的视觉生成，必须先完整执行：
`_skills/ppt-designer/prompts/02_generate_slides.md`

这里的“复用”是原文级复用，不是摘抄几条规则。
若未读取该文件，不得开始 HTML 编码。
但“原文级复用”指的是**吸收创作方法、模板完整性纪律与截图质控闭环**，不是把母版里所有泛化表述当作本次交付的硬版式义务。

优先级固定为：
1. 当前章节工作台文稿与视觉导演稿
2. 当前 deliverable 的 audience-facing / review contract
3. 母版提示词里仍然适用的方法论

若母版中的泛化规则会把页面推向“字段填空”“高密度硬塞”“为了完整而多造一层”，必须以讲课目标为准做删减，不得让母版压过当前工作台真相面。

## 1. 两种执行模式（从一开始可选）

1. 模式A：文档直驱（默认）
2. 模式B：skill 增强（调用 `complex-ppt-designer`）

两种模式都遵循同一主提示词和同一页面约束。

## 2. 输入与输出

输入：
- `<讲课目录>/大纲/<章节文件>.md`
- `<讲课目录>/大纲/<章节文件>_视觉导演稿.md`
- `<讲课目录>/参考材料/`（可选）

输出：
- `<讲课目录>/幻灯片/<章节文件>.html`
- `<讲课目录>/幻灯片/<章节文件>_视觉质控.md`

## 3. 本工作台只追加这 8 条约束

1. 禁止视觉模板路由：不得出现 `renderSlide/layoutByType/cardsGrid/pageType`
2. 每页必须在 `slidesData` 中写成独立 `content` 代码块
3. 来源/署名/页眉页码按页控制，不得全局固定
4. 严格对齐 `大纲/<章节文件>.md`：页序、标题、核心内容一致
5. 同一章节文件至少使用 3 种布局语言，避免全篇“标题+列表”
6. 页面必须保持 16:9（1152x648）且无滚动条
7. 必须落实 `大纲/<章节文件>_视觉导演稿.md` 中的全章气质、分页角色与禁退化语法
8. 没有 `*_视觉导演稿.md`，不得开始 HTML 编码

## 3.0 目标导向优先级（强制，新增）

1. 页面首先服务“把课讲好”，而不是服务“让模板看起来完整”。
2. `title / core_sentence / page_core_content` 是内容池，不是固定槽位；能并就并，能删就删，能只保留一个主命题就不要再发明第二标题层。
3. 封面页、边界页、结尾页优先做减法：不要再自动挂平台眉题、右侧辅助卡、来源胶囊、底栏说明或流程概览。
4. 任何 backstage 口吻都只能留在工作台和 review 文档里，不得借 HTML 生成之名重新变成 audience-facing chip。
5. 若某页最合理的解是保留明确留白，就允许留白；不要为了“高信息密度”把页面重新推回卡片墙。

## 3.1 导演防单调约束（强制）

1. `*_视觉导演稿.md` 不仅要写“像什么 / 不像什么”，还必须写明：
   - 节奏曲线：哪些页是峰值页、缓冲页、转折页、收束页
   - 破格页清单：哪些页必须明显区别于相邻页
   - 页面家族上限：同一家族最多连续几页、哪些组合禁止连续出现
2. 视觉导演的职责不是制造统一模板，而是用统一母题维持页间差异。
3. 若用户没有指定强隐喻风格，不得擅自把整章导向“控制台 / 手账 / 发布会”等强视觉隐喻。
4. 若任务是优化已有课件，必须先对照旧版认可稿，写清：
   - 必须保留的旧版优点
   - 禁止出现的回归（更粗糙、更单调、更重、更挤、更像统一模板页）
5. 视觉导演不负责改写内容判断，但必须保护逐页设计里已经明确的具体讲者建议、模型版本和工具名，不得在 HTML 阶段抹平成空泛表达。
6. 若连续 2-3 页都滑成“标题 + 同形卡片 / 同形舱室 / 同形对比板”，视为导演失效。
7. 关键页必须有视觉峰值，不允许整章只剩均匀、安全、可读但无记忆点的页面。

## 3.2 防偏约束（强制，新增）

1. **禁止“脚本代做视觉设计”**：不得新增或使用任何脚本把 `大纲/*.md` 机械映射为 `slidesData`（例如自动拼装固定卡片/固定流程模板）
2. **必须由大模型逐页生成**：视觉页内容必须按 `02_generate_slides.md` 逐页理解后直接写入 `幻灯片/<章节文件>.html`
3. **脚本仅可用于校验**：允许脚本做页数统计、命名检查、转换与验收，不允许脚本生成页面视觉内容
4. **若触发冲突，以主提示词优先**：只要任务是“按提示词生成 HTML”，就不得先写生成器脚本
5. **执行前闸门**：开始编码前必须显式确认“本次只改 `<title>` 和 `slidesData`，不重构模板核心脚本”
6. **导演稿前置**：开始编码前必须显式确认“已完成 `<章节文件>_视觉导演稿.md`，并将其作为 HTML 生成输入之一”
7. **公开来源口径**：公开证据页禁止写 `本目录参考材料`、`来源索引`、`内部资料` 这类内部占位来源，必须改写成听众可理解的公开来源口径
8. **复杂结构显式轨道**：中轴、判断梯、时间带、多区对照、环状/十字结构页必须显式定义网格、轨道或锚点，不得依赖自动流式排布
9. **禁止伪滚动 UI**：幻灯片中的卡片、主框、舱室内部不得出现像滚动条、滚动柄或可滚动面板一样的视觉残留
10. **如果问题属于上游 framing，不得假装是 render 微调**：一旦发现 audience-facing 纯度、讲述目标、信息层级本身出了问题，必须把 `rerun_from_stage` 回指到 `detailed_outline / slide_blueprint / visual_direction`，而不是总停在 `render_html`

## 3.3 连续讲课一致性约束（强制，新增）

1. **仅分章节任务启用跨章一致性**：当存在 `PartN` 时，才要求先观察 `PartN-1` 的视觉语言（标题层级、留白密度、脚注与署名习惯、配色节奏）
2. **无明确要求时，不新增版式元素**：前一部分没有“每页右下角署名”，后一部分不得自行新增
3. **冲突处理顺序**：用户最新明确要求 > 连续性约束 > 分章草稿里的局部写法
4. **一致性自检**：分章节任务中，每完成一个 Part，必须检查与前一 Part 是否出现突兀差异（署名、页脚、色调、字体节奏）

## 3.4 版本判断约束（强制，新增）

1. **默认单版本**：若用户未明确要求“标准版+通用版”，且当前目录不存在 `_通用版` 的 Part 文件，完整课件只产出 1 份
2. **禁止惯性双产出**：不得仅因为历史项目存在通用版策略，就在当前任务自动生成双版本完整课件
3. **合并策略按现场判断**：只有在“目录已有通用版 Part”或“用户明确要求双版本”时，才使用双标题合并
4. **命名随任务口径**：单版本任务的完整课件文件名不得强行追加“临床版/通用版”标签

## 4. HTML 输出契约（必须）

1. 保留 `#slide-display-area`
2. 保留 `#prev-btn`、`#next-btn`
3. 每页外层使用 `.slide`，非当前页使用 `.invisible`
4. `slidesData` 结构如下：

```js
const slidesData = [
  { content: `...第一页...` },
  { content: `...第二页...` }
];
```

## 4.5 渲染验收闸门（强制，新增）

> HTML 生成完成后，必须先过本闸门，再进入 PPTX 转换

1. 先确认页面能正常加载：
   - `#slide-display-area` 可见
   - `#prev-btn`、`#next-btn` 可切换
2. 确认 `slidesData` 页数与 `大纲/<章节文件>.md` 的页数一致。
3. 确认所有页面保持 16:9（1152x648）且无滚动条、无明显溢出。
4. 若发现页数不一致、页面加载失败、内容溢出，必须先回改 HTML，不得直接进入 `tool/converter.py`。
5. 只有通过本闸门后，才允许运行：

```bash
python3 tool/converter.py "<讲课目录>/幻灯片/<章节文件>.html"
```

## 4.6 AI 原生截图质控与主动回修闸门（强制）

> 通过渲染验收后，仍不得立刻转 PPTX；必须先做截图质控

1. 在浏览器中逐页切换 `<讲课目录>/幻灯片/<章节文件>.html`，完成截图。
2. 质控记录必须写入 `<讲课目录>/幻灯片/<章节文件>_视觉质控.md`。
3. 视觉质控以 AI 原生看图审阅为主判据；脚本只能做页数、加载、明显越界等保底校验。
4. 至少检查以下明显问题：
   - 页面内容溢出
   - 内容遮挡
   - 排版混乱
   - 字体拥挤或字号失衡
   - 图表/流程图裁切
   - 视觉重心偏移
   - 页间风格突兀跳变
   - 连续同构或“统一安全模板”感过强
   - 关键页没有视觉峰值、记忆点不够
5. 若任务是优化已有课件，必须加做“旧版 vs 新版”相对审阅，判断新版是否更粗糙、更单调、更重、更挤或更像统一模板页。
6. 只要命中上述明显问题，必须先回修 HTML，再重新截图/复查。
7. 未通过本闸门，不得进入 `tool/converter.py`。
