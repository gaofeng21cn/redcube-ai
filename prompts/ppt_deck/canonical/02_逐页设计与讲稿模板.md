# 逐页设计与讲稿模板（Step 3 / 3.5 专用）

> 用途：把 `详细大纲.md` 落成可直接编码的 `大纲/*.md`，并补齐进入 HTML 前必须存在的 `*_视觉导演稿.md`

## 文件命名

- 输出目录：`<讲课目录>/大纲/`
- 文件命名：
  - 单文件讲课：`讲课主题.md`
  - 分章节讲课：`PartX_主题.md`
- 配套视觉导演稿：
  - 单文件讲课：`<讲课主题>_视觉导演稿.md`
  - 分章节讲课：`PartX_主题_视觉导演稿.md`

## 单页输出模板（必须字段）

```text
幻灯片: <两位编号，例如 01>
页面类型: <封面/章节/证据/对照/机制/总结...>
标题: <本页标题>
本页目标: <解释概念/建立信任/推动决策/承上启下/...>

页面核心内容:
- 模块1：<屏幕可见文本>
- 模块2：<屏幕可见文本>
- 模块3：<屏幕可见文本>

视觉呈现方式:
- 布局：<左右/上下/三栏/中心辐射/流程图>
- 图表：<柱状图/流程图/对照卡片/时间线>
- 配色：<主色 + 辅色 + 强调色>
- 素材引用方式：<引用哪张图，如何裁剪或摆放>

角标与标识策略:
- 页眉/页码：<显示/不显示；若显示，放置位置>
- 署名：<显示/不显示；默认封面或封底显示>
- 来源文字：<显示/不显示；默认不显示，仅证据页显示>

是否需要外部公开证据:
- <需要/不需要>
- 若需要：<写明是趋势/监管/行业/公开数据中的哪一类>

证据与图源:
- 来源1：<事实资产或参考材料路径 + 图表编号/段落锚点>
- 来源2：<可选>

讲稿:
<口语化讲稿，含本页核心句与解释逻辑>

过渡句:
<与下一页衔接的一句话>
```

## Step 3 完成标准

1. 每页都有：页面核心内容 + 视觉呈现方式 + 讲稿 + 过渡句
2. 每页都知道“本页目标”是什么，不把解释页、信任页、决策页混写
3. 若标记“需要外部公开证据”，则在“证据与图源”中明确给出公开来源或对应参考材料文件
4. 每个关键数字都能回溯到“证据与图源”
5. 每个章节都已补齐配套 `*_视觉导演稿.md`
6. 仅凭 `大纲/*.md` + `*_视觉导演稿.md` 就能直接写出 HTML，不需要再猜
7. 禁止把逐页设计写成统一模板口径（例如全是“标题+三条要点”）
8. `*_视觉导演稿.md` 已明确：哪几页负责拉高节奏，哪几页负责解释缓冲，哪几页必须明显破格

## 与 Step 4 的衔接规则

1. Step 4 视觉生成按：`素材库/03_装配规则/03_视觉生成执行手册（文档直驱+Skill增强）.md`
2. 视觉生成主提示词固定复用：`_skills/ppt-designer/prompts/02_generate_slides.md`
3. 文档直驱和 skill 增强都不能绕过本模板字段
4. 没有 `*_视觉导演稿.md`，不得进入 HTML
